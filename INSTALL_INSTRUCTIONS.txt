üåê DEPLOYING TO VERCEL (Make it Live!)
================================================================================

OPTION A: Using Vercel CLI (Recommended)
-----------------------------------------

1. Install Vercel CLI:
   npm install -g vercel

2. Login to Vercel:
   vercel login
   (Follow the prompts to login)

3. Deploy your site:
   vercel
   (Answer the setup questions)

4. Deploy to production:
   vercel --prod

5. Done! You'll get a live URL like:
   https://your-portfolio.vercel.app


OPTION B: Using GitHub + Vercel Dashboard
------------------------------------------

1. Create a GitHub account (if you don't have one)
   Visit: https://github.com

2. Install Git (if not installed)
   Download from: https://git-scm.com

3. Initialize Git in your project:
   git init
   git add .
   git commit -m "Initial commit"

4. Create a new repository on GitHub
   ‚Ä¢ Go to GitHub and click "New Repository"
   ‚Ä¢ Name it "portfolio" or similar
   ‚Ä¢ Copy the repository URL

5. Push your code to GitHub:
   git remote add origin YOUR_GITHUB_REPO_URL
   git branch -M main
   git push -u origin main

6. Deploy on Vercel:
   ‚Ä¢ Go to https://vercel.com
   ‚Ä¢ Sign up/Login (use GitHub account)
   ‚Ä¢ Click "New Project"
   ‚Ä¢ Import your GitHub repository
   ‚Ä¢ Click "Deploy"

7. Done! Your site is live!

üÜò TROUBLESHOOTING
================================================================================

Problem: "npm is not recognized"
Solution: Install Node.js from nodejs.org

Problem: Port 5173 already in use
Solution: The dev server will use the next available port automatically

Problem: Dependencies won't install
Solution: 
  1. Delete node_modules folder
  2. Run: npm cache clean --force
  3. Run: npm install

Problem: Site won't load
Solution:
  1. Check if dev server is running
  2. Make sure you're using the correct URL
  3. Try restarting the dev server (Ctrl+C, then npm run dev)

Problem: Build fails
Solution:
  1. Make sure all dependencies are installed
  2. Check for error messages in terminal
  3. Try: npm run build
